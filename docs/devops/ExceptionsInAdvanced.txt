Advanced Java Topic Errors and Exceptions
A quick reference for advanced Java exceptions, threading errors, and runtime failures.

1. java.lang.IllegalMonitorStateException
   - Occurs when wait()/notify()/notifyAll() is called outside a synchronized block.
   - Common in thread coordination and lock misuse.

2. java.lang.IllegalThreadStateException
   - Happens when Thread.start() is called more than once on the same thread object.
   - The runtime reports the exact JVM exception name.

3. java.lang.InterruptedException
   - Thrown when a sleeping or waiting thread is interrupted.
   - Always handle by restoring the interrupt status or propagating the exception.

4. Deadlock
   - Not a JVM exception, but a fatal runtime hang caused by circular lock acquisition.
   - Example: Thread A waits for lock X while holding lock Y, Thread B waits for Y while holding X.

5. Race condition
   - Not an exception name; a logic error when shared mutable state is accessed without synchronization.
   - Can lead to inconsistent output, lost updates, and nondeterministic bugs.

6. java.lang.OutOfMemoryError
   - Happens when the JVM cannot allocate more heap space.
   - Typical for heavy object creation or memory leaks in advanced code.

7. java.lang.StackOverflowError
   - Thrown for recursive methods with no proper base case.
   - Often seen in deep recursion or accidental self-calling methods.

8. java.lang.NullPointerException
   - Occurs when code dereferences a null object reference in advanced algorithms.
   - Watch for null checks around shared objects and thread-shared data.
